# IntelliJ IDEA 단축키 가이드 (Mac)

> IntelliJ IDEA를 처음 사용하는 팀원을 위한 Mac 기준 디버깅 + 자주 쓰는 단축키 정리

---

## 1. 디버그 모드 실행

| 단축키 | 설명 |
|--------|------|
| `^ + D` | 디버그 모드로 실행 (현재 설정) |
| `^ + R` | 일반 실행 (Run) |
| `^ + Shift + D` | 현재 파일/메서드를 디버그 실행 |
| `^ + Shift + R` | 현재 파일/메서드를 일반 실행 |

> **팁**: Spring Boot 앱은 `TriagainApplication`의 main 메서드 왼쪽 ▶ 버튼 → Debug로도 시작 가능

---

## 2. 브레이크포인트

| 단축키 | 설명 |
|--------|------|
| `⌘ + F8` | 현재 줄에 브레이크포인트 토글 |
| `⌘ + Shift + F8` | 브레이크포인트 목록 보기 / 조건 설정 |

### 브레이크포인트 종류

- **라인 브레이크포인트**: 가장 기본. 해당 줄에서 실행 멈춤
- **조건부 브레이크포인트**: 브레이크포인트 우클릭 → Condition에 조건식 입력 (예: `crewId.equals("abc123")`)
- **예외 브레이크포인트**: `⌘ + Shift + F8` → `+` → Java Exception Breakpoints → 특정 예외 클래스 지정
- **메서드 브레이크포인트**: 인터페이스 메서드에 걸면 실제 구현체에서 멈춤 (헥사고날에서 유용)

---

## 3. 디버깅 중 코드 이동

| 단축키 | 설명 |
|--------|------|
| `F8` | Step Over — 현재 줄 실행 후 다음 줄로 (메서드 안으로 안 들어감) |
| `F7` | Step Into — 메서드 내부로 진입 |
| `Shift + F8` | Step Out — 현재 메서드를 빠져나가 호출부로 복귀 |
| `⌥ + F9` | Run to Cursor — 커서 위치까지 실행 |
| `F9` | Resume — 다음 브레이크포인트까지 실행 계속 |
| `⌘ + F2` | 디버그 세션 종료 (Stop) |
| `⌥ + F8` | Evaluate Expression — 디버깅 중 임의 코드 실행 |

### Step 계열 요약

```
F8  → 한 줄 실행 (안으로 안 들어감)
F7  → 한 줄 실행 (메서드 안으로 들어감)
⇧F8 → 현재 메서드 빠져나감
⌥F9 → 커서까지 쭉 실행
F9  → 다음 브레이크포인트까지 쭉 실행
```

## 4. 디버깅 중 값 확인

| 방법 | 설명 |
|------|------|
| **Variables 탭** | 디버그 창 하단에서 현재 스코프의 모든 변수 확인 |
| **Watches** | Variables 탭 → `+` 버튼으로 감시할 표현식 추가 |
| **마우스 호버** | 코드 위에 마우스를 올리면 현재 값이 툴팁으로 표시 |
| `⌥ + F8` | **Evaluate Expression** — 임의 표현식을 즉석에서 실행하고 결과 확인 |
| **인라인 힌트** | 디버깅 중 코드 옆에 변수 값이 회색으로 자동 표시 |

> **Evaluate Expression 활용 예시**:
> - `crew.getMembers().size()` — 현재 크루 멤버 수 확인
> - `challenge.isActive()` — 챌린지 활성 상태 확인
> - `LocalDateTime.now()` — 현재 시간 확인

## 5. 실전 디버깅 흐름 예시

"크루 참여 시 정원 초과 에러가 발생하는 상황"을 디버깅하는 예시:

```
1. JoinCrewService.execute()에 브레이크포인트 설정      ⌘ + F8
2. 디버그 모드로 앱 실행                                ^ + D
3. API 호출 → 브레이크포인트에서 멈춤
4. crew 객체의 현재 멤버 수 확인                        Variables 탭
5. crew.isFull() 메서드 내부로 진입                     F7
6. maxMembers와 현재 members.size() 비교 확인          마우스 호버
7. 조건 확인 후 메서드 빠져나오기                        Shift + F8
8. 다음 줄로 이동하며 흐름 추적                          F8
9. 원인 파악 후 디버그 종료                              ⌘ + F2
```

## 6. 디버깅 꿀팁

### 조건부 브레이크포인트

특정 조건에서만 멈추고 싶을 때:
1. 브레이크포인트 **우클릭**
2. Condition 필드에 조건식 입력
3. 예: `member.getUserId().equals("user-123")`

### 로깅 브레이크포인트 (Non-Suspending)

코드를 멈추지 않고 로그만 남기고 싶을 때:
1. 브레이크포인트 우클릭
2. **Suspend** 체크 해제
3. **Evaluate and log** 체크 → 표현식 입력
4. 예: `"크루 멤버 수: " + crew.getMembers().size()`

### Drop Frame

잘못 Step Into 했을 때 한 프레임 되돌리기:
- 디버그 창의 Call Stack에서 되돌릴 프레임 우클릭 → **Drop Frame**
- 주의: 부수효과(DB 저장 등)는 되돌릴 수 없음

### 원격 디버깅

서버에 디버거 연결:
1. Run → Edit Configurations → `+` → Remote JVM Debug
2. 서버 JVM 옵션에 `-agentlib:jdwp=transport=dt_socket,server=y,suspend=n,address=*:5005` 추가
3. IntelliJ에서 해당 설정으로 Debug 실행

| `⌘ + F` | 현재 파일에서 텍스트 검색 |
| `⌘ + R` | 현재 파일에서 텍스트 치환 |

## 8. 코드 네비게이션

| 단축키 | 설명 |
|--------|------|
| `⌘ + B` / `⌘ + Click` | 선언부로 이동 (Go to Declaration) |
| `⌘ + ⌥ + B` | 구현체로 이동 (Go to Implementation) — 인터페이스 → 구현 클래스 |
| `⌘ + U` | 상위 메서드/클래스로 이동 (Super Method) |
| `⌥ + F7` | 사용처 찾기 (Find Usages) |
| `⌘ + F7` | 현재 파일 내 사용처 하이라이트 |
| `⌘ + [ / ]` | 이전/다음 위치로 이동 (뒤로/앞으로) |
| `⌘ + L` | 특정 줄 번호로 이동 |
| `⌘ + F12` | 현재 파일의 구조 보기 (메서드/필드 목록) |
| `⌥ + ↑ / ↓` | 메서드 단위로 이동 |
| `F2` / `Shift + F2` | 다음/이전 에러로 이동 |

### 헥사고날 아키텍처에서 유용한 네비게이션

```
UseCase 인터페이스 → ⌘ + ⌥ + B → Service 구현체
Port 인터페이스   → ⌘ + ⌥ + B → Adapter 구현체
Service 메서드    → ⌥ + F7    → Controller에서 호출하는 곳
```

## 10. 코드 생성 & 자동완성

| 단축키 | 설명 |
|--------|------|
| `⌘ + N` | Generate — 생성자, getter, setter, equals, hashCode, override 등 자동 생성 |
| `⌥ + Enter` | **Quick Fix** — 에러/경고에 대한 수정 제안 (가장 많이 쓰는 단축키!) |
| `⌘ + ⌥ + Enter` | 현재 줄 위에 새 줄 삽입 |
| `⌘ + Shift + Enter` | 구문 완성 (세미콜론, 중괄호 자동 추가) |
| `^ + Space` | 기본 자동완성 |
| `^ + Shift + Space` | 스마트 자동완성 (타입 기반 추천) |
| `⌘ + P` | 메서드 파라미터 정보 보기 |
| `⌘ + J` | Live Template 삽입 (sout, psvm 등) |

### 자주 쓰는 Live Template

| 약어 | 확장 결과 |
|------|-----------|
| `sout` | `System.out.println()` |
| `soutv` | `System.out.println("변수 = " + 변수)` |
| `psvm` | `public static void main(String[] args)` |
| `fori` | `for (int i = 0; i < ; i++)` |
| `iter` | Enhanced for loop |

## 13. TOP 10 필수 단축키

매일 쓰는 단축키만 모았습니다. 이것만 외워도 생산성이 크게 올라갑니다.

| 순위 | 단축키 | 설명 | 사용 빈도 |
|------|--------|------|-----------|
| 1 | `⌥ + Enter` | Quick Fix — 뭐든 빨간 줄 뜨면 이것부터 | 매 분 |
| 2 | `⌘ + B` | 선언부로 이동 | 매 분 |
| 3 | `⌘ + ⌥ + B` | 구현체로 이동 (인터페이스 → 구현) | 수시로 |
| 4 | `Shift + Shift` | Search Everywhere — 뭐든 찾을 때 | 수시로 |
| 5 | `⌘ + Shift + F` | 프로젝트 전체 텍스트 검색 | 수시로 |
| 6 | `Shift + F6` | Rename — 이름 한번에 바꾸기 | 자주 |
| 7 | `⌘ + ⌥ + L` | 코드 포맷팅 | 저장 전 |
| 8 | `⌘ + F8` | 브레이크포인트 토글 | 디버깅 시 |
| 9 | `F8 / F7` | Step Over / Step Into | 디버깅 시 |
| 10 | `⌘ + [ / ]` | 뒤로/앞으로 이동 | 탐색 시 |
